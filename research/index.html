Deterministic MFA Trust Score Generation

This project redesigns MFA trust-score generation to be more realistic, explainable, and security-driven. Instead of relying on excessive randomness, the system follows deterministic rules: every user begins with low trust, legitimate users can build trust over time, and attacker sessions are intentionally constrained to low-trust behavior.

Project Overview

The goal of this work is to make synthetic MFA trust data more realistic and more useful for attacker detection by replacing arbitrary randomness with repeatable, security-motivated rules.

What This Research Focuses On

This project explores deterministic trust-score generation for multi-factor authentication. The generator was redesigned so trust progression follows understandable rules, improving interpretability while still preserving a small set of difficult cases that simulate ambiguous or malicious behavior.

Why the Redesign Matters

Overly random synthetic data can create patterns that do not reflect realistic user behavior. A deterministic approach makes the data easier to reason about, easier to audit, and better aligned with the kinds of trust signals an MFA system would actually use.

Trust scores range from 0.0 to 1.0. Legitimate users are allowed to gain trust over time, while attacker sessions are kept in a low-trust region with only a narrow overlap band to reflect edge cases without losing separability.

Core Design Decisions

These are the main rule changes that shape the current generator and make the model behavior easier to explain.

Low Initial Trust

Every user begins with low trust. This creates a cautious starting point and allows trust to be earned through consistent, expected behavior rather than assumed by default.

Legitimate Trust Growth

Normal users can build trust across sessions when their behavior remains stable. That growth models credibility accumulation instead of assigning strong trust randomly.

Constrained Attacker Band

Attacker sessions are biased toward low trust, with occasional higher-trust samples injected only to represent realistic edge cases. This preserves a clear decision margin for classification.

Risk Signals That Lower Trust

1

Unexpected location

Sessions outside expected regions are treated as higher risk.

2

Unexpected device or OS

Trust drops when session context no longer matches the user’s normal profile.

3

Unexpected IP / unusual timing

Weekend activity, unfamiliar IPs, and failed logins all contribute to lower trust.

Location-Based Overseas Feature

A major addition in this version is the geographic rule that automatically lowers trust for sessions that fall outside expected U.S. regions.

How It Works

The generator defines expected U.S. geographic regions using latitude and longitude bounds. If a generated session falls outside those regions, the trust score is capped at 0.20, reflecting a conservative security policy for out-of-region access.

Travel Injection

To keep the dataset realistic, a small portion of sessions simulate occasional travel. Around 5% of sessions are moved away from the user’s home location so the model sees some variation without letting travel dominate the dataset.

The overseas feature is intentionally simple. It adds a meaningful policy signal, but it is not meant to represent a full geolocation or travel-history model.

Model Evaluation

The generated dataset is used to train and evaluate an SVM classifier that separates legitimate and malicious sessions.

Classifier

A Support Vector Machine is trained on the resulting trust-aware dataset to learn a boundary between legitimate users and attackers.

Performance

Performance is measured with Area Under the ROC Curve. In the current setup, the model reaches 0.93 AUC, which surpasses the project target of 0.90.

Next Steps

The current generator improves realism and interpretability, but there is still more room to expand evaluation and stress-testing.

Add EER Reporting

Future evaluation will include Equal Error Rate to better understand the false positive / false negative tradeoff.

Larger-Scale Testing

Running broader experiments on a GPU environment will make it easier to compare parameter settings and alternative models.

External Dataset Comparison

A future direction is evaluating against external risk-based authentication datasets to better assess realism and generalization.
